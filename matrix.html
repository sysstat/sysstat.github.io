Matrix of activities
The following table lists all the possible activities for sar:
  • the option that should be used with sar to display the corresponding activity is given in column #3 ;
  • the option that should be used with sadc to collect the corresponding activity is given in column #4. The value "(default)" means that the activity is always collected by sadc and that no specific option is needed for that ;
  • whether SVG graphs can be created by sadf for the corresponding activity (this list will be updated as SVG support improves).
Activity Report name Option to display with sar Option to collect with sadc Available as SVG graphics Notes
Paging activity A_PAGE -B (default) Yes
I/O and transfer rate A_IO -b (default) Yes
Disks activity A_DISK -d -S DISK Yes sadc: -S XDISK also collects activity
Partitions activity A_DISK -d -S XDISK Yes sar: Activity displayed only when data read from a file
Filesystems activity A_FS -F -S XDISK Yes sar: -F MOUNT displays mountpoint names instead of filesystem names
Hugepages activity A_HUGE -H (default) Yes
Interrupts activity A_IRQ -I -S INT No
Instantaneous CPU clock frequency A_PWR_CPU -m CPU -S POWER Yes
Fans speed A_PWR_FAN -m FAN -S POWER Yes Need the sensors library (lm_sensors)
Weighted CPU clock frequency A_PWR_FREQ -m FREQ -S POWER No
Voltage inputs A_PWR_IN -m IN -S POWER Yes Need the sensors library (lm_sensors)
Devices temperature A_PWR_TEMP -m TEMP -S POWER Yes Need the sensors library (lm_sensors)
Plugged USB devices A_PWR_USB -m USB -S POWER No
Battery power A_PWR_BAT -m BAT -S POWER Yes
